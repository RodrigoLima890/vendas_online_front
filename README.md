## Explição sobre cookies
No momento estou tratando o login de usuario em que no backend após a validação das credenciais é criado um access token que esta sendo salvo em um cookie
### Tipos
#### LAX:
O cookie somente será enviado em um contexto diferente se for usado em um verbo HTTP GET
#### Strict
O cookie será enviado no mesmo contexto, independente do verbo HTTP utilizado
#### None
será enviado sem restrições em diferentes contextos
### O que é same site
é onde definimos o tipo do cookie pode ser usada para evitar ataques de CSRF(falsificação de solicitação de sites)
